Lesson 4: Deployment and Scaling Strategies

Goal: Learn how to deploy and scale chatbot backend services.

Concept Explanation

Deployment has two parts:

  1. Client Distribution: Packaging, updates, and platform compatibility (if applicable).
  2. Backend Scaling: Handling concurrent users, AI workloads, and data storage.

Backend scaling:

  • Horizontal scaling: Add more service instances behind load balancer.
  • Containerization: Docker + Kubernetes for orchestration.
  • Serverless: AWS Lambda, Cloud Functions for event-driven workloads.
  • Database scaling: Read replicas, connection pooling, caching.

AI-specific scaling:

  • LLM provider scaling: Multiple API keys, rate limit management.
  • Vector database scaling: Sharding, indexing optimization.
  • Cache layer: Redis for frequent queries and session data.

Worked Example

Kubernetes deployment for backend:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: chat-service
spec:
  replicas: 3
  selector:
    matchLabels:
      app: chat-service
  template:
    metadata:
      labels:
        app: chat-service
    spec:
      containers:
      - name: chat
        image: chat-service:latest
        ports:
        - containerPort: 3000
        env:
        - name: DATABASE_URL
          valueFrom:
            secretKeyRef:
              name: db-secret
              key: url
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /health
            port: 3000
          initialDelaySeconds: 30
          periodSeconds: 10

Auto-scaling configuration:

apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: chat-service-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: chat-service
  minReplicas: 3
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70

Exercise

Design a deployment strategy for a chatbot that:

  1. Supports 10,000 concurrent users.
  2. Handles AI response generation with <200ms latency.
  3. Scales automatically during peak hours.
  4. Provides zero-downtime deployments.

Define the infrastructure, scaling policies, and monitoring.

Answer Key

Infrastructure:

  • Backend: Kubernetes cluster with 3 node pools:
    • Chat service: 3-10 replicas (CPU-based scaling)
    • AI service: 2-8 replicas (GPU-based scaling for local models)
    • API gateway: 2-4 replicas (request-based scaling)
  • Data: PostgreSQL (primary + 2 read replicas), Redis cluster (3 nodes)
  • AI: Multiple LLM provider keys with circuit breaker fallback

Scaling policies:

  • Chat service: Scale up when CPU >70% or request latency >100ms
  • AI service: Scale up when queue depth >100 or response latency >500ms
  • Database: Read replicas when read CPU >60%, connection pooling with PgBouncer

Monitoring:

  • Prometheus metrics: request rate, error rate, latency percentiles
  • Grafana dashboards: System health, AI performance, user engagement
  • Alerting: PagerDuty for critical issues, Slack for warnings

Zero-downtime deployments:

  • Rolling updates with max surge 1, max unavailable 0
  • Health checks with graceful shutdown
  • Database migrations with backward compatibility
