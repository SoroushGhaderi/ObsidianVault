Lesson 3: AI/NLP Integration for Chatbot Intelligence

Goal: Learn how to integrate language models and retrieval systems for intelligent chatbot responses.

Concept Explanation

Modern chatbots use three AI layers:

  1. Language Model (LLM): Generates human-like responses (OpenAI, Ollama, local models).
  2. Retrieval-Augmented Generation (RAG): Fetches relevant knowledge from documents/FAQs.
  3. Conversation Memory: Maintains context across messages and sessions.

Key components:

  • Prompt Engineering: Structured prompts with system instructions, context, and user input.
  • Vector Database: Stores document embeddings for semantic search (pgvector, Pinecone).
  • Streaming: Token-by-token response delivery for perceived performance.
  • Function Calling: LLM can call external tools (search, calculations, APIs).

Architecture patterns:

  • Orchestrator Pattern: AI service coordinates LLM, retrieval, and memory.
  • Pipeline Pattern: Sequential processing: query rewriting → retrieval → generation.
  • Agent Pattern: LLM decides which tools to use dynamically.

Worked Example

RAG pipeline for support chatbot:

class SupportChatAI:
    def __init__(self, llm, vector_store, memory):
        self.llm = llm
        self.vector_store = vector_store
        self.memory = memory
    
    async def generate_response(self, user_message, session_id):
        # 1. Retrieve conversation context
        history = await self.memory.get_history(session_id)
        
        # 2. Rewrite query for better retrieval
        rewritten = await self.llm.generate(
            f"Rewrite this support question for knowledge base search: {user_message}"
        )
        
        # 3. Retrieve relevant documents
        docs = await self.vector_store.search(rewritten, top_k=3)
        
        # 4. Generate response with context
        prompt = self._build_prompt(user_message, docs, history)
        response = await self.llm.generate_stream(prompt)
        
        # 5. Store in memory
        await self.memory.add_message(session_id, user_message, response)
        
        return response
    
    def _build_prompt(self, message, docs, history):
        return f"""You are a helpful support chatbot. Answer based on the provided documentation.

Documentation:
{chr(10).join([doc.content for doc in docs])}

Conversation history:
{self._format_history(history)}

User question: {message}

Answer:"""

Exercise

Design a support chatbot AI system that:

  1. Handles multi-turn conversations with context.
  2. Retrieves from a knowledge base of 10,000 support articles.
  3. Streams responses token-by-token.
  4. Falls back to generic responses when knowledge base has no relevant info.

Define the data flow, prompt structure, and error handling.

Answer Key

Data flow:

  1. User message → Query rewriting (LLM)
  2. Rewritten query → Vector search (top 5 results with scores)
  3. Results below threshold → Fallback response generation
  4. Results above threshold → Context-augmented prompt
  5. Prompt → LLM streaming → Token buffer → SSE to client
  6. Complete response → Memory storage (async)

Prompt structure:

System: You are a support assistant. Use ONLY the provided documentation. If unsure, say "I don't have that information."

Context from knowledge base:
[Article 1: Title - Content]
[Article 2: Title - Content]

Conversation:
[User]: Previous question
[Assistant]: Previous answer

[User]: Current question

Error handling:

  • Vector search timeout → Use cached results or generic response
  • LLM timeout → Return partial response with apology
  • Memory failure → Continue without history (stateless)
  • Rate limiting → Queue with position indicator
