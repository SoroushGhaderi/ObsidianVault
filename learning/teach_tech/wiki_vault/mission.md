# Mission: Operate and Understand a Personal Wiki Vault

## Why
Learn to operate the V3 vault confidently, understand why its boundaries exist, and use Codex or another LLM without allowing the knowledge base to drift.

## Success looks like
- Explain the roles of `raw/`, `wiki/`, `journal/`, and `log/`.
- Run and interpret each command from the vault root.
- Ingest a technical source into connected source, entity, and concept pages.
- Retrieve an answer through index → hub → page traversal.
- Record an architectural decision and inspect its Git diff.
- Run linting and distinguish safe automatic fixes from judgment calls.

## Constraints
- Use `/Users/soroush/Documents/ObsidianVault` as the working Obsidian vault.
- Learn through small exercises on the real vault.
- Preserve raw evidence and avoid automatic Git commits.

## Out of scope
- Building a full semantic/vector search system.
- Large-scale vault migration or taxonomy redesign.
